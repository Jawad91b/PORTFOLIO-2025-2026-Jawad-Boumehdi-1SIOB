2. Création de la Base de Données et des Tables

J'ai créé la base nommée cinema_mission. J'ai défini les types de données selon le besoin (INT pour les identifiants et nombres, VARCHAR pour les textes, DATE pour les dates). Pour la table de jointure casting, j'ai défini une clé primaire composite (IDENT_FILM + IDENT_ACTEUR).

3. Contraintes d'intégrité

J'ai configuré les clés étrangères pour lier les tables entre elles :

  • Un film est lié à un réalisateur et un pays.
  • Le casting lie un film et un acteur.

4. Insertion des données

J'ai utilisé des requêtes SQL INSERT INTO pour ajouter les données des annexes en respectant l'ordre pour ne pas violer les contraintes (d'abord les pays et réalisateurs, puis les films).

5. Requêtes SQL (Bonus)

Film > 90mn sortis entre 2008 et 2012

SELECT TITRE FROM film WHERE DUREE > 90 AND DATE_SORTIE BETWEEN '2008-01-01' AND '2012-12-31';

Résultat : AVATAR

Acteurs ayant tourné entre 40 et 90 films

SELECT NOM, PRENOM FROM acteur WHERE NB_FILM BETWEEN 40 AND 90;

Résultat : Duris, Ford, Fisher, Weaver, Reno...
